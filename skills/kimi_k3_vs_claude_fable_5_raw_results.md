# Kimi K3 for Algorithmic Trading Research using MCP and Jesse Trade

## Description
This skill demonstrates how to leverage the Kimi K3 open-source AI model for advanced algorithmic trading research. It covers setting up the Kimi CLI, defining complex trading strategy prompts, and utilizing the Jesse Trade framework with MCP for automated strategy generation, backtesting, validation, and optimization.

## Steps

1.  **Install Kimi CLI tool.**
2.  **Log in to your Kimi account via the CLI.**
3.  **Switch the active model to Kimi K3.**
4.  **Define a detailed prompt for the AI agent**, specifying the trading pair (e.g., ETH/USDT), strategy type (e.g., trend-following with Bollinger Bands, squeeze), base indicator, performance metrics (e.g., Sharpe ratio > 1.5), risk management (e.g., risk per trade < 3%), and validation requirements (statistical significance, optimization, Monte Carlo simulations).
5.  **Configure the MCP (Multi-Channel Protocol) server**, potentially by running `/mcp config` and selecting a global scope for broader accessibility, ensuring it is running and accessible by the AI agent.
6.  **Run the Kimi agent with the prompt**, allowing it to iterate until criteria are met.
7.  **Approve permissionless execution (YOLO mode)** for continuous agent operation, understanding the associated risks.
8.  **Monitor the AI's progress** in generating, validating, and optimizing trading strategies.
9.  **Utilize the Jesse Trade framework** for strategy development, backtesting, optimization, rule testing, and Monte Carlo simulations.
10. **Review the generated reports**, including backtest results, statistical significance tests, optimization data, and Monte Carlo simulation outputs.
11. **Utilize the provided URLs** to examine the detailed backtest and simulation results.
12. **Monitor agent progress and outputs** within the Zed code editor's terminal.
