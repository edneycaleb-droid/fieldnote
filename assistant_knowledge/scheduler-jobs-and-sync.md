## Scheduler Jobs — authoritative from source code

All six jobs run in background threads managed by APScheduler:

| Job ID | Interval | Description |
|---|---|---|
| `enhance` | Every **6 hours** | DCA Enhancer — re-extracts existing skills with fresh AI pass to improve quality |
| `sync` | Every **24 hours** | Full library sync — pushes all skills + brain graph + README to GitHub |
| `watchlist` | Every **1 hour** | Processes queued YouTube URLs from the watchlist |
| `discover` | Every **2 hours** | GitHub Discovery Agent — searches repos and extracts skills |
| `code_sync` | Every **10 minutes** | Source-code-only push (app.py, agents/, templates/, fieldnote_mcp/ source files) |
| `integrations` | Every **30 minutes** | Integration Agent — health-checks providers, auto-detects new Replit Secrets, generates suggestions |

A **file watcher** also triggers `code_sync` within ~16 seconds of any file change in the workspace.

## GitHub Sync — what is and is not pushed

### Pushed (source files)
- Top-level: `app.py`, `main.py`, `replit.md`, `pyproject.toml`, `.gitignore`, `PROVIDERS.md`
- Directories: `agents/`, `templates/`, `fieldnote_mcp/` (source files only — see exclusions below)

### Pushed (knowledge)
- `fieldnote_skills/*.md` — via `sync_skill()` called after every `_save_skill()`
- `_brain.json` — via `sync_brain()` after every enhance/discovery run
- `README.md` — auto-generated by `_build_readme()` from the skill index

### Never pushed (excluded by name or directory)
- Secrets/keys: `local_keys.json`, `.env`, `.replit`, `replit.nix`
- Runtime state: `integration_agent_status.json`, `integration_dynamic.json`, `status.json`
- Replit-specific: `mcp_config.json`, `install.log`
- Caches: `__pycache__/`, `fieldnote_repos/`, `fieldnote_skills/` (managed separately via sync_skill)

## Transcription Pipeline (caption-first)

The system tries each method in order and stops at the first success:
1. **YouTube captions** — `youtube-transcript-api`; instant, zero compute, free; only works if video has a captions track
2. **Groq Whisper** — `whisper-large-v3` via Groq API; cloud, ~8h audio/day free; used when captions unavailable
3. **faster-whisper** — local CPU inference; always free; final fallback; models cached at `.cache/fw_models/`

## Provider Router Order

For all LLM calls: Groq → Gemini → OpenAI → HuggingFace → OpenRouter

Current models (as of 2026-07-16):
- Groq: `llama-3.3-70b-versatile`, `gemma2-9b-it`, `mixtral-8x7b-32768`
- Gemini: `gemini-2.0-flash`, `gemini-1.5-flash-8b`
- OpenAI: `gpt-4o-mini`, `gpt-4o`
